设计模式： 

> 设计模式是指在**软件开发过程**中，**经过验证的**，用于解决在**特定环境**下，**重复出现**的，**特定问题**的解决方案。

理解C++的内存模型（继承和组合）
下面是继承的例子：

```cpp
class Subject : public Base  
{
};
```
组合的例子

```cpp
class Subject
{
private:
	Base base;
};

// 第二种组合例子
class Subject
{
private:
	Base* base;
};
```
下面是上面中的例子在内存中的情况：
继承和使用对象进行组合在内存中的情况是分布在一起的，使用指针进行组合，可以子类和父类指针分开，从而可以实现运行时的多态（也叫**晚绑定**）

## 模式设计的原则：
1. 依赖倒置原则
2. 开放封闭原则
3. 面向接口编程
4. 封装变化点
5. 单一职责原则
6. 里氏替换原则
7. 接口隔离原则
8. 对象组合优于继承
